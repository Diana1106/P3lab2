Introduccion

Inicios y finalidad de la empresa bonnex creadora de Dolphin red social media Bonnex es un apasionado equipo de desarrollo de software con finalidad de brindar un asistente de desarrollo de software para plataformas de redes sociales. Desde sus inicios, se han dedicado a la creación de herramientas que unen a las personas, ayudando a cientos de miles de aspirantes a web masters, el equipo de bonnex noto lo innecesariamente complicado o demasiado limitante que es crear una red social desde cero: la mayoría de las soluciones requieren habilidades técnicas o no le permiten personalizar el sitio.




Para que sirve bonnex Dolphin Entonces en 2000 se creó la primera versión de Dolphin red social media para facilitar la creación de plataformas de redes sociales al brindar al web master una herramienta para manejar todo lo relacionado con el establecimiento de una comunidad en línea con un potencial de personalización casi ilimitado: un sistema para personas con ideas. Quienes utilizan Dolphin Web master aspirantes que buscan convertir sus ideas en redes sociales activas.

¿Qué es Dolphin? El sistema de administración de la comunidad más completo, desarrollado y
ampliamente respaldado con enfoque en redes sociales, intercambio de contenido y participación social móvil.

Recomendaciones

  • Una cuenta de alojamiento compartido puede ser suficiente para poner en marcha un sitio,
    pero usted tendrá que actualizar rápidamente a un servidor dedicado o VPS cuando su sitio se vuelve popular.

  • asegúrese de que su servidor web está configurarse de acuerdo con los requisitos del sistema de Dolphin
    y que tiene toda la información para hacerlo.

  • al tener mantener un red social activa y muy frecuentado Dolphin tiende a responder lento,
    elegir un buen servidor es una alternativa muy interesante si llegas a tener este problema.
  • Los servidores mas recomendamos para la buena eficiencia en Dolphin normalmente son de paga.

    Tecnologias que ofrece dolphin para desarrollar plataformas de redes sociales

  • Compartir contenido multimedia: Fotos y videos de los eventos de la empresa, así como archivos de audio se pueden cargar y difundir mediante esta plataforma.
  • Chat: Para agilizar la comunicación interna y ahorrar tiempo se puede establecer un chat tanto de texto y video llamadas entre varios usuarios.
  • Subir archivos: Se pueden subir archivos relevantes para la operación diaria para ponerlos a disposición de los empleados para su descarga y visualización.
  • Foros de discusión: Establecer un espacio para compartir ideas u opiniones de los miembros del equipo, así como poder dividir los temas por categorías, herramientas para la moderación, etc.
  • Perfiles: Cada integrante de la empresa tiene su propio perfil y tiene la opción de personalizarlo para hacer a la red más atractiva.
  • Eventos: Se establecen fechas, confirmación de asistencia, cargar fotos relacionadas con la actividad, foros de discusión dentro del evento, entre otras funciones.
  • Noticias: Publicar los hechos más relevantes, anuncios importantes y alertas en el momento o programar la fecha donde se mostrará con anticipación.

    • Requisitos de alojamiento de bonnex Dolphin

    Una instalación predeterminada de Dolphin funciona con las configuraciones más comunes del servidor de alojamiento, sin embargo, ya que Dolphin es un paquete con múltiples scripts y aplicaciones, es muy importante asegurarse de que se cumplen todos los siguientes requisitos. La mayoría de los problemas técnicos se producen debido a una configuración incorrecta del servidor, así que asegúrese de que su proveedor de alojamiento sea compatible con Dolphin.

    Siempre que el servidor esté configurado correctamente, es posible ejecutar Dolphin en una cuenta de hospedaje compartido. Sin embargo, recomendamos que tenga un VPS o un servidor dedicado para Dolphin.

    NOTA: Dolphin se puede instalar con un solo clic a través de Softaculous -http://www.softaculous.com/softwares/socialnetworking/Dolphin

    Requisitos del servidor

    Linux/Unix (Red Hat, Debian, FreeBSD, Mandrake, etc.) o el sistema operativo Windows Nota: Kernel de Linux 2.6.32 o superior es necesario para ffmpeg.exe Servidor web Apache v.1.3 y superior o LiteSpeed. Otros servidores web también se pueden utilizar, pero se requieren reglas de reescritura personalizadas, tenemos uno para el servidor web nginx. PHP 5.4.0 o superior, incluyendo PHP 7 (PHP 5.3-5.6 para Dolphin 7.2-7.3, al menos PHP 5.2.0 para Dolphin 7.1.x). PHP-CLI (versión de línea de comandos) también debe ser instalado, y debe ser configurado de la misma manera y tener las mismas extensiones requeridas por Dolphin que la versión DSO/CGI.

    Si se instala un módulo de seguridad (como mod_security para Apache), debe poder configurarse para carpetas específicas (o deshabilitarlas). mod_rewrite para Apache Debe poder establecerse una conexión directa a boonex.com (es decir, la conexión sin necesidad de autorización de proxy). Esto es necesario para la verificación de la licencia.




    Configuración recomendada del servidor Dado que Dolphin es una plataforma que consume muchos recursos, es muy importante "afinar" su servidor. En algunos casos (para sitios populares) es posible que tenga que configurar servidores en clúster y/o un servidor de base de datos remoto. El software alternativo de "servidor web", como nginx, también puede ser beneficioso para el rendimiento de su sitio. Consulte Recomendaciones de configuración del servidor de alojamiento para obtener más información sobre cómo puede ajustar el servidor.

    En la segunda página de la instalación, deberá indicar las rutas necesarias que requiere el script. Aquí están:

    Instalar script - Paso 3 - Base de datos

    En la tercera página especificará la información de la base de datos:

    Archivo SQL : esta es la ruta relativa al archivo de creación de la base de datos, que se determina por defecto. Si su archivo reside en un directorio diferente, debe indicar su ruta relativa aquí. La carpeta base de la ruta se considera la carpeta de instalación de Dolphin.
    Base de datos vacía : elija "sí" o "no". Elegir "sí" vaciará toda la base de datos, es decir, eliminará sus tablas. Piénselo dos veces antes de elegir esta opción. La opción "no" es la opción predeterminada y no causará daños a su base de datos.
    Tablas vacías : elija "sí" o "no". Elegir "sí" vaciará todas las tablas en la base de datos mientras mantiene intacta la estructura de las tablas. La opción predeterminada es "sí" y no debe realizar ningún cambio aquí.
    Nombre de host de la base de datos : aquí debe indicar el nombre de host de su base de datos, que generalmente es 'localhost'. Consulte las notas que hizo durante la creación de la base de datos.
    Nombre de la base de datos, usuario de la base de datos y contraseña de la base de datos : aquí debe indicar el nombre de la base de datos y los requisitos de usuario que creó en el Paso 2 . Nuevamente, consulte las notas que tomó al crear la base de datos y el usuario.
